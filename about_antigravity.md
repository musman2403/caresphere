## How AI "Credits" & Tokens Work
Most modern AI models don't use a feature-by-feature credit system (e.g., "1 credit for backend logic, 2 credits for animations"). Instead, usage is billed or measured in **Tokens**.

- **What is a Token?** A token is a chunk of text or code (roughly 4 characters in English). 
- **Input vs. Output Cost:** You use "Input Tokens" whenever you send me files, prompts, and context. You use "Output Tokens" for every piece of code or text I generate in response.
- **Cost Scaling:** Tasks like writing complex **backend database logic**, implementing intricate **GSAP animations**, or building **responsive UI components** do not have individual price tags. Instead, their "cost" directly correlates to the volume of code required. The longer and more intricate the code output, the more output tokens are consumed.

## Choosing the Right AI Model for Your Task
Different models are optimized for different workflows. Working efficiently often means picking the right model for the job:

1. **Flagship / "Heavyweight" Models (e.g., Gemini 3.1 Pro, GPT-4o, Claude 3.5 Sonnet)**
   - **Best for:** Complex algorithmic problem solving, designing distributed backend architectures, agentic coding over an entire repository, and hunting down convoluted bugs.
   - **Characteristics:** Highly accurate, massive context windows, but use expensive/premium compute capability.

2. **Fast / "Lightweight" Models (e.g., Gemini 1.5 Flash, Claude 3 Haiku, GPT-4o-mini)**
   - **Best for:** Rapid prototyping, simple script writing, standard code completions, writing boilerplate code, or formatting documentation. 
   - **Characteristics:** Lightning-fast response times and highly token-efficient, perfect for tasks that don't require deep logical reasoning.

3. **Specialized Asset & Image Models (e.g., Imagen 3, Stable Diffusion, Runway)**
   - **Best for:** Generating mock UI assets, placeholders for web applications, or generating video/audio transitions.
